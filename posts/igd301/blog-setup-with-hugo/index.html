Setting up the website#

Install#

To set up the website, I started by downloading the Hugo binary. Then I used a command line tool (I use Git Bash) to create a new site, with the following command:

hugo new site quickstart

This automatically created the following hierarchy in the folder I ran the command into (except for the git files, that I added myself for the deployment):

Choosing a theme#

To make the website look the way I wanted it to, I chose a premade theme, among the many available. I chose the PaperMod one, as I liked it and it looked well documented. I downloaded it and put the files in the “themes” subfolder of my website.

Configuring the whole website#

I used the config.yml file, at the root of my website folder, to customize it further :

  • I changed the home page : title, description, etc.
  • I added links to subsections in the home page.
  • I modified global variables, like the boolean showNavLinks, which defines whether it is possible or not to press an arrow to go from one blog post to another.

Testing#

To test the website before it is really deployed, I can run a local Hugo server using the following command:

hugo server -D

Then I can just visualize the state of by website by going to http://localhost:1313 in any Web browser.

# Setting up the website

## Install
To set up the website, I started by downloading the Hugo binary. Then I used a command line tool (I use Git Bash) to create a new site, with the following command:

```hugo new site quickstart```

The content itself is written in Markdown, which allows me to easily add sections and subsections, as well as embedding images, hyperlinks or code snippets.

I can use the variables provided by the Paper Mod theme to customize the appearance of my website. For instance, the “cover” variable allows me to add a cover image in a blog post.

In Hugo, it is also possible to create subsections of the website. To do so, I had to create one subfolder of the “content” folder for each subsection. This subfolder must then contain an _index.md file, which contains the title and description of the subsection. Here is the hierarchy and the _index.md file of the IGD301 subsection.

Deploying with GitHub#

In order to make the website accessible for everyone, I had to serve the static files (HTML, CSS…) from a webserver. Thankfully GitHub provides a free static site hosting with GitHub Pages.

To reduce the hassle of building the website locally, and to reduce the risk of losing the original code, I created two distinct repositories:

  • A private one containing the unbuilt code
  • A public one with the static web files (its name has to be <user>.github.io)

Deployment workflow#

With every push to the private GitHub repository, thanks to the workflow.yml file, the following chain of action is triggered:

  • A container (provided by GitHub Actions) fetches the unbuilt code and installs Hugo
  • The container builds the website in the ./public folder
  • The container pushes all these static files from ./public to the public repository
  • GitHub updates the static files served under https://thanaell.github.io
